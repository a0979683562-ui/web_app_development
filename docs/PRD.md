# 產品需求文件 (PRD) - 食譜收藏系統

## 1. 專案概述
- **背景與動機**：希望能有一個專屬的地方紀錄自己喜歡的食譜，隨時可以新增、尋找與回顧，避免忘記或找不到曾經做過的美味料理。
- **目標用戶**：我自己（單一使用者導向的個人工具）。
- **核心價值主張**：提供一個簡單、直覺的個人化食譜管理空間，讓烹飪變得更有條理與樂趣。

## 2. 功能需求
系統需具備以下主要功能：

- **功能一：食譜圖文紀錄**
  - 使用者故事：作為系統使用者，我希望可以新增包含圖片與文字說明的食譜，以便清楚記錄食材與料理步驟。
- **功能二：標籤分頁**
  - 使用者故事：作為系統使用者，我希望可以替食譜加上分類標籤（如：中式、甜點、快速料理等），並透過分頁瀏覽，以便有系統地整理與尋找食譜。
- **功能三：關鍵字搜尋**
  - 使用者故事：作為系統使用者，我希望可以輸入關鍵字搜尋食譜標題或內容，以便快速找到我想做的特定料理。
- **功能四：我的最愛**
  - 使用者故事：作為系統使用者，我希望可以將特定的食譜標記為「我的最愛」，以便未來能更快速地存取常做的料理。
- **功能五：烹飪筆記**
  - 使用者故事：作為系統使用者，我希望可以在食譜下方或個人專區中加上烹飪筆記，以便記錄每次實作的改良心得或失敗經驗、隨時調整口味。

## 3. 非功能需求
- **技術限制**：
  - 框架：Flask
  - 前端樣板引擎：Jinja2
  - 資料庫：SQLite
- **效能與安全考量**：
  - 系統效能：確保基本的資料讀寫與搜尋反應快速，頁面載入流暢。
  - 資料保存：雖然是單機或個人伺服器使用，但建議實作或保留手動備份 SQLite 資料庫檔案的方法。
  - 畫面響應：介面須支援響應式設計（RWD），在手機上方便邊看邊煮。防範基本的 XSS 及 SQL Injection 攻擊。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須具備核心功能)**：
  - 食譜圖文的 CRUD (新增、讀取、更新、刪除)。
  - 關鍵字搜尋功能。
- **Should Have (強烈建議具備)**：
  - 標籤系統（可對食譜加標籤及依照標籤過濾分頁）。
  - 烹飪筆記紀錄功能。
- **Nice to Have (加分項目)**：
  - 我的最愛標記（快速篩選清單）。
  - 將食譜輸出成 PDF 或分享連結的功能。
